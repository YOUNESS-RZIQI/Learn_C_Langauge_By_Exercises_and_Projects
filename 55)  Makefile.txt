 What is a Makefile?

A Makefile is a special file used by the `make` command to automatically compile and manage programs.  
It tells the compiler **what to build**, **how to build it**, and **when to rebuild it**.

 Why we need it:
- To avoid typing long `gcc` commands every time.  
- To recompile only files that changed (saves time).  
- To organize large projects with many `.c` files.  
- To automate building, cleaning, and rebuilding the program.

 Example of a simple Makefile:
-------------------------------------------------
CC = gcc
CFLAGS = -Wall -Wextra -Werror
SRC = main.c utils.c
OBJ = $(SRC:.c=.o)
NAME = my_program

$(NAME): $(OBJ)
	$(CC) $(CFLAGS) $(OBJ) -o $(NAME)

clean:
	rm -f $(OBJ)

fclean: clean
	rm -f $(NAME)

re: fclean $(NAME)
-------------------------------------------------

- `CC` ‚Üí the compiler (gcc).  
- `CFLAGS` ‚Üí the compiler flags for warnings or options.  
- `SRC` ‚Üí list of source files.  
- `OBJ` ‚Üí same files but with `.o` extension.  
- `NAME` ‚Üí the name of the executable file.  
- `clean` ‚Üí deletes object files.  
- `fclean` ‚Üí deletes everything (object files + executable).  
- `re` ‚Üí rebuilds everything from scratch.

- `make re` ‚Üí cleans and recompiles everything.
